La [Singularity Platform](https://www.sentinelone.com/platform/) se basa en **See** (visibilidad), **Protect** (defensa) y **Resolve** (respuesta con IA) en endpoint, nube e identidad. Abajo mapeamos esos pilares a nuestro stack soberano.

### Pilares Singularity ↔ stack soberano

| Pilar | Singularity | En nuestro stack |
|-------|-------------|-------------------|
| **See** | Visibilidad en toda la empresa | Atabey Command Center, estado de nodos/servicios, Wazuh (logs/alertas), `estado-final-sistema.json` |
| **Protect** | Defensa en endpoint, nube, identidad | Wazuh (atabey_guardian), Nginx PM (gateway), Security Fortress, auth y roles, secret scanning |
| **Resolve** | Respuesta autónoma e IA en el ecosistema | Procedimientos documentados, backups (`sovereign-backup.sh`), scripts de respuesta (panic/listener) |

Ciberseguridad 101 incluye: Seguridad en la nube, Datos e IA, Endpoints, Identidad, Servicios (SOC, MDR, DFIR), Inteligencia sobre amenazas, XDR y Ciberseguridad (DevSecOps, parches, filtración de datos). Para gobierno estatal/local: **[SEGURIDAD-GOBIERNO-ESTATAL-Y-LOCAL.md](./SEGURIDAD-GOBIERNO-ESTATAL-Y-LOCAL.md)**.

## Mapa: categorías 101 ↔ stack soberano

| Categoría 101 | Conceptos clave | En nuestro stack |
|---------------|-----------------|-------------------|
| **Seguridad en la nube** | CNAPP, CWPP, responsabilidad compartida | Docker (Atabey Fortress), Nginx PM (gateway), escaneo de secretos. |
| **Cloud Data Security** | Detección en almacenamiento, respuesta automatizada, local y conforme | Nextcloud cifrado (Vault), backups, [CLOUD-DATA-SECURITY.md](./CLOUD-DATA-SECURITY.md), `scripts/cloud-data-security-check.sh`. |
| **Datos e IA** | SIEM, análisis de seguridad, data lakes | Wazuh (SIEM) en `atabey_guardian`, datos censales en Baserow, backups soberanos. |
| **Seguridad de endpoints** | EPP, EDR, gestión de terminales | Security Fortress (`/platform/security-fortress.html`), monitoreo de servicios, health checks. |
| **Seguridad de identidad** | Confianza cero (ZTA), PAM, IAM, mínimo privilegio | Auth/sesión en plataforma, login soberano, roles (admin/operator/citizen). |
| **Servicios** | SOC, MDR, threat hunting, DFIR | Wazuh para detección y logs; scripts de backup y respuesta (Atabey panic/listener). |
| **Inteligencia sobre amenazas** | MITRE ATT&CK, threat hunting, honeypots | Wazuh reglas y alertas; documentación de fuentes (p. ej. `docs/FUENTES-OFICIALES-JUSTICIA-GLOBAL.md`). |
| **XDR** | EDR vs XDR, MXDR | Visibilidad unificada vía Atabey Command Center, estado de nodos y servicios. |
| **Ciberseguridad** | DevSecOps, parches, filtración de datos, políticas | Secret scanning (Gitleaks/TruffleHog), backups encriptados, documentación de estado final. |

## Checklist rápido (alineado a 101)

- [ ] **Secretos:** Ejecutar `./scripts/secret-scan.sh` y revisar hallazgos; CI ya ejecuta Gitleaks/TruffleHog en push/PR.
- [ ] **Censo:** Baserow (:8080) con base "Ierahkwa Sovereign Registry" y columnas definidas.
- [ ] **Bóveda:** Nextcloud (:8081) con carpeta "Grandfather's Archive" y cifrado de servidor activado.
- [ ] **Guardian:** Wazuh (`atabey_guardian`) en marcha; revisar logs y alertas.
- [ ] **Gateway:** Nginx Proxy Manager (:81) configurado (proxy inverso, HTTPS donde aplique).
- [ ] **Backup:** `sovereign-backup.sh` en cron y, si aplica, `SOVEREIGN_BUNKER_PATH` para copia offline.
- [ ] **Cloud Data Security:** Nextcloud cifrado; ejecutar `./scripts/cloud-data-security-check.sh` para verificar último backup (ver [CLOUD-DATA-SECURITY.md](./CLOUD-DATA-SECURITY.md)).
- [ ] **SMB / negocio:** Ejecutar `./scripts/smb-security-check.sh` para checklist rápido (ver [SMB-CYBERSECURITY.md](./SMB-CYBERSECURITY.md)).
- [ ] **Identidad:** Políticas de acceso y roles (admin/operator/citizen) documentadas y aplicadas en la plataforma.
