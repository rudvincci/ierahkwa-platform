# Plan de atractivo de inversión por token

**Gobierno Soberano de Ierahkwa Ne Kanienke**  
**Objetivo:** Que **cada token IGT** tenga un **caso de inversión claro** en su pre-launch y whitepaper, de modo que los inversionistas quieran invertir.

Complementa: `docs/PLAN-NEGOCIOS-TOKENS-PRE-LAUNCH-WHITEPAPER.md`.

---

## 1. Objetivo

- Cada token debe responder de forma explícita: **por qué un inversionista debería interesarse en este token**.
- El mensaje debe estar en la **página de pre-lanzamiento** (visible de inmediato) y desarrollado en el **whitepaper** (sección "Caso de inversión" o equivalente).
- Sin depender de terceros: todo el argumento de valor es propio (soberanía, ecosistema ISB/BDET, utilidad real).
- **Negocios separados:** Cada token tiene su propia página de negocio independiente; solo en el main dashboard se mezcla todo; en exchanges/trading cada uno por su cuenta. Todos comparten el mismo backend y data. Ver `docs/PRINCIPIO-NEGOCIOS-SEPARADOS-DASHBOARD-UNICO.md`.

---

## 2. Qué debe tener cada token para atraer inversión

### 2.1 En la página pre-launch (index.html)

Debe incluir **al menos**:

| Elemento | Descripción | Ejemplo |
|----------|-------------|--------|
| **Propuesta de valor en una línea** | Frase que resume por qué este token tiene valor. | "IGT-MD: el token que monetiza la seguridad soberana y los contratos de defensa dentro del ecosistema Ierahkwa." |
| **Por qué invertir** | Bloque corto (3–5 bullets): demanda, uso real, respaldo, ventaja frente a alternativas. | • Respaldo BDET / soberanía • Uso en contratos reales • Staking con yield • Liquidez en TradeX |
| **Oportunidad** | Qué gana el holder: staking, descuentos, gobernanza, fees, acceso. | "Stake IGT-XX y recibe X% en IGT-REWARD" o "Pago de servicios con descuento para holders" |
| **Call to action** | Botones visibles: Comprar / Stake / Ver whitepaper / Unirse al launchpad. | Enlace a TradeX, BDET, whitepaper, citizen-launchpad |

### 2.2 En el whitepaper (sección "Caso de inversión")

Cada whitepaper debe incluir una sección **"Caso de inversión"** (o "Razones para invertir") con:

| Sección | Contenido mínimo |
|---------|------------------|
| **Propuesta de valor** | Qué problema resuelve el token y por qué tiene valor dentro del ecosistema soberano. |
| **Demanda y uso** | Quién usa el token (gobierno, empresas, ciudadanos, partners), en qué productos/servicios y volumen potencial si se puede estimar. |
| **Cómo gana valor el token** | Mecanismos que hacen que el token sea deseable: staking con recompensas, quema de fees, uso obligatorio en un servicio, gobernanza, acceso exclusivo, descuentos. |
| **Respaldo y seguridad** | Emisión bajo Gobierno Soberano, blockchain ISB, banco BDET, cumplimiento BHBK. Por qué es más confiable que un token sin respaldo soberano. |
| **Roadmap breve** | 3–5 hitos (ej. Fase 1: listing TradeX, Fase 2: staking, Fase 3: integración con departamento X). |
| **Riesgos y mitigaciones** | 2–4 riesgos (volatilidad, regulación, adopción) y qué hace el emisor para mitigarlos. |
| **Disclaimer** | El documento no es asesoría de inversión; el inversionista asume sus propios riesgos. |

Cada token puede pegar y completar esta plantilla en su `whitepaper-es.md` (y equivalentes en otros idiomas):

### Riesgos y mitigaciones
| Riesgo | Mitigación |

*Este documento no constituye asesoría legal, fiscal ni de inversión. El inversionista asume sus propios riesgos.*

## 4. Ideas por categoría de token (para rellenar cada caso)

Cada token puede inspirarse en su categoría para redactar el caso de inversión:

| Categoría | Ángulo para inversionistas |
|-----------|----------------------------|
| **Gobierno / Ministerios** | Respaldo soberano, uso en contratos y servicios públicos, estabilidad por demanda institucional. |
| **Finance (STABLE, GOV, STAKE, etc.)** | Respaldo BDET, liquidez, yield por staking, uso como colateral y en DeFi. |
| **Comercio / Trade / Exchange** | Fees compartidos, liquidez, volumen de TradeX y partners. |
| **Casino / Gaming / Lotto** | Parte de los fees o premios vinculados al token; ecosistema de entretenimiento soberano. |
| **Servicios (Health, Edu, Travel, etc.)** | Demanda real de ciudadanos y partners; descuentos para holders; integración con plataformas existentes. |
| **Tech (AI, VPN, Cloud, etc.)** | Acceso a servicios premium con el token; revenue share o quema de fees. |
| **Identity / Legal / Vote** | Necesidad institucional; uso en KYC, votaciones, contratos; valor por adopción soberana. |

## 5. Checklist por token (inversionistas quieran invertir)

Para cada token, verificar:

- [ ] **Pre-launch:** Hay una frase de "propuesta de valor" visible.
- [ ] **Pre-launch:** Hay bloque "Por qué invertir" (3–5 bullets).
- [ ] **Pre-launch:** Se explica qué gana el holder (staking, descuentos, gobernanza, etc.).
- [ ] **Pre-launch:** Hay botones CTA: Trade / Stake / Whitepaper / Launchpad (según corresponda).
- [ ] **Whitepaper:** Existe sección "Caso de inversión" con: valor, demanda, cómo gana valor, respaldo, roadmap, riesgos.
- [ ] **Whitepaper:** Incluye disclaimer de no asesoría de inversión.

## 6. Dónde poner cada cosa

| Contenido | Dónde |
|-----------|--------|
| Propuesta de valor en una línea | Pre-launch (index.html) — encabezado o primera card. |
| Por qué invertir (bullets) | Pre-launch — card dedicada "Por qué invertir en IGT-XX". |
| Botones CTA | Pre-launch — ya existen Trade/Liquidity/Wallet; añadir "Whitepaper" y, si aplica, "Stake" o "Launchpad". |
| Caso de inversión completo | Whitepaper (whitepaper-es.md y otros idiomas) — sección fija. |
| Roadmap / riesgos | Whitepaper; opcionalmente un resumen en pre-launch. |

## 8. Resumen

- **Objetivo:** Que cada token tenga un **caso de inversión** claro en pre-launch y whitepaper.
- **Pre-launch:** Propuesta de valor, "Por qué invertir", qué gana el holder, CTAs (Trade, Stake, Whitepaper, Launchpad).
